# Техническое задание (курсовой проект)

## Проект
Интернет-магазин кроссовок (Django + DRF).

## Роли и права

### 1) Администратор
1. Управление товарами: создание, редактирование, удаление.
2. Управление заказами: просмотр всех заказов, изменение статусов.
3. Управление пользователями и ролями.
4. Модерация отзывов.

### 2) Покупатель
1. Просмотр каталога товаров, фильтрация по цене/категории/производителю.
2. Управление корзиной (добавление, изменение количества, удаление).
3. Оформление заказа с адресом доставки.
4. Просмотр только своих заказов и статусов.
5. Оставление отзывов только после покупки.

## Бизнес-логика и валидации
Реализованы следующие проверки:
1. Проверка остатков при добавлении в корзину.
2. Проверка остатков при оформлении заказа.
3. Валидация формата адреса доставки.
4. Проверка минимальной/максимальной суммы заказа (500–100000 руб).
5. Проверка доступа к заказам (владелец или администратор).
6. Валидация отзывов (отзыв только после покупки товара).

## Оптимизация запросов
Используются:
- `select_related` для связей `Order -> user/status`, `Review -> user/product`, `CartItem -> product/cart`, `Product -> supplier`.
- `prefetch_related` для коллекций `Order -> items`, `Product -> categories/images`, `Cart -> items`.

## Сериализаторы
1. Используется `SerializerMethodField` (например, вычисляемые поля в товарах и отзывах).
2. Используется передача данных через `context` (например, `request`, `product`).

## Аннотации
Реализованы аннотированные поля:
1. `avg_rating` — средний рейтинг товара.
2. `sold_quantity` — количество проданных пар.
3. `favorites_count` — количество добавлений в избранное.

## Фильтрация
Реализован `ProductFilterSet`:
- `min_price`, `max_price`
- `category`
- `manufacturer`
- `is_active`

## Тестирование
Добавлены API-тесты по ключевым требованиям:
- валидация корзины и заказа,
- доступ к заказам,
- валидация отзывов,
- фильтрация товаров,
- проверка аннотации `favorites_count`.
